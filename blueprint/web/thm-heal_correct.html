8 Heal Move Overflow

The heal check fails at HP gaps of 255, 511, and 767 due to a broken 16-bit comparison.

[Buggy Heal Check] PokeredBugs.healCheckFails Models cp [hl]; sbc [hl]; jp z. Only checks the low-byte subtraction result.

[Fails at Gap 255] PokeredBugs.heal_fails_at_gap_255 def:heal_check \(\texttt{maxHP} = 300\), \(\texttt{currentHP} = 45\) (gap \(= 255\)): move incorrectly fails.

[No False Fail Below 256] PokeredBugs.no_false_fail_below_256 def:heal_check When \(\texttt{maxHP} \le 255\), the check is correct: only fails at full HP.

[Bad Gaps Enumerated] PokeredBugs.bad_gaps_enumerated def:heal_check For \(\texttt{maxHP} \le 999\): false failures occur only at gaps 255, 511, or 767.

[Correct Check] PokeredBugs.correct_no_false_fail A correct 16-bit equality check never has false failures.
