1 SM83 CPU Model

A Lean 4 formalization of the Sharp SM83 CPU, the processor powering the original Game Boy. The model covers registers, flags, memory, and \(\sim \)40 opcodes with semantics faithful to the hardware specification.

1.1 Core Types

Definition 1.1 CPU State
✓
#

Lean declarations

  • SM83.CPUState

The SM83 CPU state: eight 8-bit registers (\(A\), \(F\), \(B\), \(C\), \(D\), \(E\), \(H\), \(L\)), 16-bit \(\mathit{SP}\) and \(\mathit{PC}\), a total memory function \(\texttt{BitVec}\ 16 \to \texttt{BitVec}\ 8\), and a halt flag.

Definition 1.2 Default State
✓
#

Uses

  • Definition 1.1

Lean declarations

  • SM83.defaultState

All registers zero, \(\mathit{SP} = \mathtt{0xFFFE}\), \(\mathit{PC} = \mathtt{0x0100}\), memory all zeros.

1.2 Flag Register

Definition 1.3 mkFlags
✓
#

Lean declarations

  • SM83.mkFlags

Build a flag byte from individual booleans (\(Z\), \(N\), \(H\), \(C\)). Bits 3–0 are always zero, matching the SM83 hardware constraint.

Theorem 1.4 Flag Low Nibble Invariant
✓
#

Uses

  • Definition 1.3

Lean declarations

  • SM83.mkFlags_low_nibble_zero

For all flag combinations: \(\texttt{mkFlags}(z,n,h,c) \mathbin {\& } \mathtt{0x0F} = 0\).
